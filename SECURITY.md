# Security Policy

## 报告安全问题

如果发现仓库中存在真实密钥、内部服务地址、客户数据、生产配置或其它敏感信息，请通过 GitHub Security Advisory 或私下联系维护者，不要在公开 issue 中粘贴敏感内容。

## 不应提交的内容

- API key、token、client secret、私钥、证书。
- 真实账号、邮箱验证码、OAuth callback secret。
- 内部服务域名、生产数据库连接串、schema 细节。
- 客户数据、生产日志、截图中的隐私信息。
- 私有部署 URL、预发/生产环境变量文件。

## 维护者处理原则

一旦确认敏感信息泄漏，应立即：

1. 删除公开内容并重写相关历史或撤销发布。
2. 轮换所有可能受影响的密钥和凭据。
3. 在 changelog 或 advisory 中说明影响范围，不重复暴露敏感值。
4. 增加或更新扫描规则，避免同类问题再次出现。
